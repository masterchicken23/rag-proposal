# Vectorization

Uses Google Embedding-001

Vectors make it possible to measure similarity between any two pieces of content, capturing intent and context

# Agentic Framework
Single-Agent, Voice-First – Powered by Gemini Live API
User Voice → Wake Word ("Hey Bruno") → Context Retrieval → Gemini Live Session → Real-time Audio Response
Explanation
Gemini Live API handles the full conversation loop natively over a persistent WebSocket –  context is injected once at session start, and the model manages its own turn history.

# Inputs
What is injected into model at session initialization
System prompt (persona, rules, end triggers)
Conversation memory (Gemini manages turns natively)
Top-Ranked Manual Chunks (retrieved from ChromaDB)
Machine ID (scopes session to specific asset) + Live audio stream

# Session Storage
Persistent, scoped, and token-aware
A sliding window trims older turns to stay within token limits, always preserving the most recent diagnostic steps
Explanation
PostgreSQL over Redis or in-memory – session data for a troubleshooting workflow is relational by nature: a session belongs to a technician, references a machine, and contains an ordered sequence of turns.

# Outputs
Voice Response
Real-time PCM audio streamed back turn-by-turn. 
Interrupted immediately if the user speaks.
Structure Graph + Solution Manual
Rendered as an interactive troubleshooting canvas — each node maps to a diagnostic step, cause, or action.
